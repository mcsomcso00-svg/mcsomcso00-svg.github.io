# 0 點擊頁面該合併、刪除還是補料？決策流程

**一個 90 天 0 點擊的頁面只有三種處理方式：主題有價值但和其他頁重複 → 合併；主題沒人搜也沒轉換價值 → 移除；主題對但內容太空 → 補真材料。** 判斷順序是先問「這個主題值不值得存在」，再問「該由哪一頁來承載它」。

## 為什麼一定要三選一，不能放著？

因為 0 點擊頁面有持續成本。它佔用 Googlebot 的爬取配額（Google 官方文件明確說低價值 URL 影響 crawl budget）、稀釋內部連結權重，而且在整站品質評估中扮演扣分項。實務經驗上，一個 300 頁的部落格若有 200 頁長期 0 點擊，處理掉這 200 頁後，剩餘頁面的索引速度與排名通常在 4–8 週內可觀察到改善。

先用 GSC 拉出清單（操作見〈[GSC 三個地方揪出內容負債](/zh-hant/blog/gsc-content-debt/)〉），然後對每一頁跑下面的流程。

## 決策流程：四個問題

依序回答四個問題，答案會把你導向三條路之一：

1. **這個主題有人搜、或對轉換有幫助嗎？**
   - 都沒有 → **移除**。
   - 有 → 下一題。
2. **站上有沒有另一頁在打同一個（或更大的）主題？**
   - 有，而且那頁表現更好 → **合併**進那一頁。
   - 沒有 → 下一題。
3. **這頁的內容有具體資訊嗎？**（數據、步驟、案例、價格——任何 AI 寫不出來的東西）
   - 沒有，只是「文章形狀的空殼」 → **補真材料**。
   - 有 → 下一題。
4. **內容具體但還是 0 點擊？**
   - 檢查是不是標題／meta 不吸引人（有曝光無點擊）、或關鍵字選錯（連曝光都沒有）。這屬於優化問題，不是負債處理。

## 路線一：合併（consolidate）

適用：多篇薄文打同一主題、互相蠶食，或一篇舊文的內容可以併進支柱頁。

操作步驟：

1. 選定「存活頁」——通常是曝光較高、外鏈較多、或層級較高的那一頁。
2. 把被併頁面中**還有價值的段落**（獨特數據、案例、FAQ）搬進存活頁，不是全文貼上。
3. 被併頁面設 **301 轉址**到存活頁。注意：301 要指向「主題最相關」的頁，不要全部丟給首頁——Google 把大量指向首頁的 301 當成 soft 404 處理。
4. 更新站內所有指向舊 URL 的內部連結，直接改連存活頁（不要依賴 301 跳轉）。
5. 從 sitemap 移除舊 URL。

轉址細節與常見錯誤，見〈[網站搬家不掉排名的 301 轉址清單](/zh-hant/blog/301-redirect-checklist/)〉。

## 路線二：移除（remove）

適用：主題本身沒有搜尋需求、沒有外鏈、沒有轉換角色——例如過期活動頁、測試頁、自動生成的標籤頁。

操作步驟：

1. 確認沒有外部連結指向它（GSC 的「連結」報表或任何外鏈工具）。有外鏈的頁面改走合併路線，把連結權重 301 到相關頁。
2. 刪除頁面，讓伺服器回 **410 Gone**（比 404 讓 Google 更快移出索引；404 也可以，只是慢一點）。
3. 清掉站內指向它的內部連結與 sitemap 條目。
4. 不需要用 GSC 的「移除網址」工具——那是暫時遮蔽用的，不是刪頁流程的一部分。

## 路線三：補真材料（refill）

適用：主題對、意圖對、站上也只有這一頁在打，但內容是空殼。這是三條路裡**唯一會增值**的一條，也最花力氣。

「真材料」指的是 AI 憑空寫不出來、競爭對手懶得放的東西：

- **真實案例**：你實際做過的專案、踩過的坑、before/after 數字。
- **價格區間**：這件事外包要多少錢、工具月費多少——使用者最想知道、大家最不敢寫。
- **規格與限制**：什麼情況下不適用、上限在哪裡。
- **客戶會問的問題**：把業務信箱裡真實出現過的問題做成 FAQ。

補料後記得更新 `date_updated`、用網址檢查工具請求重新檢索，並從相關文章加 1–2 條內部連結進來。

## 優先順序：先修最礙事的 30 頁

不要試圖一口氣處理 200 頁。按「曝光量 × 主題重要性」排序，先處理 30 頁，觀察 2–4 週，再處理下一批。**在負債清完之前，暫停新增內容**——往漏水的桶子加水沒有意義。

## 常見問題（FAQ）

**Q1：刪除或合併後，多久看得到效果？**
索引層面 2–6 週（舊 URL 移出索引、轉址生效）；排名與爬取效率層面通常 1–3 個月。變化幅度取決於負債佔比——負債越重，清完後改善越明顯。

**Q2：301 轉址要保留多久？可以之後拿掉嗎？**
Google 官方建議至少保留一年，讓訊號完整轉移。實務上如果沒有技術負擔，永久保留最安全，因為外部可能還有舊連結。

**Q3：補料和重寫有什麼差別？**
補料是在既有 URL 上增加具體資訊，保留原本累積的索引歷史；重寫如果連 URL 都換，等於砍掉重練。除非原文方向完全錯誤，否則優先在原 URL 上補料。

**Q4：頁面才發布一個月就 0 點擊，要處理嗎？**
不用。90 天是最低觀察窗口，新頁面給它一季的時間；季節性主題再拉長到一年。
