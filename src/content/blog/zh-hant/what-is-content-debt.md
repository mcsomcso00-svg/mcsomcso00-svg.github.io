# 內容負債是什麼？90 天 0 點擊頁面正在拖垮你

**內容負債（content debt）是指網站上「已發布、但長期不產生點擊或轉換」的頁面——最典型的判定標準是 90 天內自然搜尋 0 點擊。** 這些頁面不是中性的存在：它們消耗 crawl budget、稀釋站內權重、拉低 Google 對整個網域的品質評價。

## 為什麼「放著不管」比你想的更貴

很多站長的直覺是：「這些頁面又沒害處，放著說不定哪天會有流量。」實務上剛好相反。Google 在官方文件（Search Central 的 crawl budget 說明）明確指出，**低價值 URL 會對網站的爬取與索引產生負面影響**。而在多數內容網站的 GSC 資料裡，常見的分布是：**前 10% 的頁面貢獻超過 80% 的自然點擊**，剩下一長條尾巴幾乎是零。

當一個站有 500 頁、其中 300 頁 90 天 0 點擊時，Google 看到的是一個「大部分內容沒人需要」的網站。這會影響三件事：

1. **爬取排程**：Googlebot 把時間花在爬沒人看的舊頁，新文章反而排隊等索引。
2. **權重分配**：內部連結把 PageRank 平均灑進一堆死頁，賺錢頁分到的變少。
3. **整站品質訊號**：自 Helpful Content 系統併入核心演算法後，Google 是以「整站」為單位評估內容品質，一批空殼頁會拖累好頁面。

## 內容負債的四種常見型態

| 型態 | 特徵 | 常見來源 |
|---|---|---|
| 空殼文 | 有標題、有段落結構，但沒有任何具體資訊 | AI 批量產文未經人工補料 |
| 重複／蠶食頁 | 多篇文章打同一個關鍵字，互相搶排名 | 沒有主題群集規劃就亂寫 |
| 過期內容 | 講的活動、價格、版本早已失效 | 新聞式、促銷式文章 |
| 孤兒頁 | 站內沒有任何連結指向它 | 改版後遺留、標籤頁、測試頁 |

## 怎麼判定一頁是不是內容負債？

用 Google Search Console 的成效報表即可，不需要付費工具：

1. 開啟 **成效 → 搜尋結果**，日期設「過去 3 個月」。
2. 切到「網頁」分頁，依點擊數由小到大排序。
3. 點擊 = 0 且曝光極低（例如 < 50）的頁面，就是第一批嫌疑犯。
4. 再對照 **索引 → 網頁** 報表裡的「Crawled - currently not indexed」與「Discovered - currently not indexed」——出現在這兩區的頁面，代表 Google 已經用行動告訴你：它不覺得值得收錄。

詳細的三步驟操作流程，見〈[GSC 三個地方揪出內容負債](/zh-hant/blog/gsc-content-debt/)〉。

## 找到之後怎麼辦？三選一

每一頁 0 點擊頁面，只有三條路：

- **合併**：主題有價值但內容太薄、或和其他頁重複 → 併入主題頁或支柱頁，原網址 301 轉址過去。
- **移除**：主題本身就沒人搜、也沒有轉換價值 → 直接刪除（回 410 或 404），並清掉指向它的內部連結。
- **補真材料**：主題對、意圖對，只是內容空 → 補上真實案例、價格區間、規格限制、客戶實際會問的問題，讓它從空殼變成有血有肉。

決策流程與每條路的操作細節，見〈[0 點擊頁面該合併、刪除還是補料？](/zh-hant/blog/zero-click-pages/)〉。

## 「先修最礙事的 30 頁，再新增內容」

這是處理內容負債最重要的順序原則。多數人發現流量停滯時的反射動作是「再多寫一點」，結果是往一個漏水的桶子裡加水。正確做法：

1. 從 GSC 匯出 90 天 0 點擊頁面清單。
2. 按「曝光量 × 主題重要性」排序，挑出**最礙事的 30 頁**——通常是曝光有一點但點擊掛零、又剛好卡在你重要關鍵字上的頁面。
3. 這 30 頁逐一執行合併／移除／補料。
4. 處理完、觀察 2–4 週索引與排名變化後，才開始新增內容。

每季重跑一次這個循環，內容負債就不會再堆積。

## 常見問題（FAQ）

**Q1：刪掉一堆頁面，網站規模變小，排名會不會反而掉？**
不會因為「頁數變少」而掉。Google 評估的是品質而非數量；實務上大規模剪枝（content pruning）後，剩餘頁面的平均排名經常上升，因為爬取資源與內部權重都更集中。前提是刪的確實是沒流量、沒外鏈、沒轉換的頁。

**Q2：90 天 0 點擊是絕對標準嗎？**
不是，它是一個實用的起點門檻。季節性內容（例如報稅、節慶）要拉長觀察窗口到一年；剛發布不滿 90 天的新文章也不算負債。

**Q3：頁面有曝光但 0 點擊，算內容負債嗎？**
算「輕度負債」，但處理方式不同——有曝光代表 Google 認得這頁的主題，問題多半出在標題與 meta description 不吸引人，先改寫這兩項再觀察，不必急著刪。

**Q4：內容負債和 GEO（AI 引用）有關係嗎？**
有。AI 引擎挑選引用來源時同樣偏好高信任網站；一個充滿空殼頁的站，被 ChatGPT 或 Perplexity 引用的機率也更低。清負債是 SEO 和 GEO 的共同地基。
