# GSC 三個地方揪出內容負債（附操作）

**要找出網站的內容負債，只需要 Google Search Console 的三個地方：成效報表（篩 90 天 0 點擊頁）、網頁索引報表（看 Crawled / Discovered - currently not indexed 兩個狀態）、網址檢查工具（逐頁確認 Google 怎麼看它）。** 全程免費，30 分鐘可以完成一輪稽核。

## 開始前：你需要什麼

只需要一個已驗證的 GSC 資源（property），且網站上線超過 90 天——因為我們的核心判定標準就是「90 天 0 點擊」。2026 年的 GSC 介面成效資料保留 16 個月，足夠做季度稽核與年度回顧。

## 第一站：成效報表——揪出 90 天 0 點擊頁

這是最重要的一站，實際操作步驟：

1. 左側選單點 **「成效」→「搜尋結果」**。
2. 上方日期篩選器改成 **「過去 3 個月」**（90 天）。
3. 下方表格切換到 **「網頁」** 分頁。
4. 點「點擊」欄位標題，由**小到大**排序。
5. 右上角 **「匯出」→ Google 試算表**，把整份清單拉出來。

在試算表裡加一欄「判定」，用兩個條件分類：

- **點擊 = 0、曝光 < 50**：重度負債。Google 幾乎不給這頁任何機會，走合併或移除。
- **點擊 = 0、曝光 ≥ 50**：輕度負債。Google 有讓它出現在搜尋結果，但沒人想點——通常是標題與描述的問題，優先改寫〈[標題標籤與 meta description](/zh-hant/blog/title-meta-writing/)〉再觀察。

注意一個陷阱：GSC 的網頁分頁預設最多顯示 1,000 列。大站要用「網頁」篩選器分目錄（例如 `/blog/`）分批匯出，才不會漏掉長尾。

## 第二站：網頁索引報表——Google 已經投票的頁面

左側選單點 **「索引」→「網頁」**（Page indexing 報表），往下捲到「網頁未編入索引的原因」清單。重點看兩個狀態：

| 報表狀態 | Google 的意思 | 對你的意義 |
|---|---|---|
| **Crawled - currently not indexed**（已檢索，目前未編入索引） | 我爬了，讀完了，決定不收錄 | 品質判決：內容不值得進索引，是最明確的負債訊號 |
| **Discovered - currently not indexed**（已找到，目前尚未編入索引） | 我知道這頁存在，但不急著爬 | 爬取優先度太低——整站品質或內部連結出了問題 |

操作：點進每個狀態可以看到 URL 範例清單（同樣可匯出）。把這兩份清單和第一站的 0 點擊清單交叉比對——**同時出現在「0 點擊」和「Crawled - currently not indexed」的頁面，就是最該優先處理的核心負債**。

如果「Discovered - currently not indexed」的數量佔比很高（例如超過已索引頁的 30%），問題通常不在單頁而在整站：crawl budget 被浪費、或內部連結結構讓 Google 覺得這些頁不重要。這時要回頭看〈[crawl budget 是什麼](/zh-hant/blog/what-is-crawl-budget/)〉。

## 第三站：網址檢查工具——逐頁確診

對前兩站篩出的重點頁面（建議先處理 30 頁），用最上方的**網址檢查（URL Inspection）**逐一貼入 URL，確認三件事：

1. **「網址在 Google 服務中」還是「不在」**——確認索引狀態與報表一致。
2. **檢索日期**——上次爬取是幾個月前？越久沒爬，代表 Google 越不在乎這頁。
3. **參照網頁（Referring page）**——Google 是從哪裡發現這頁的？如果來源是奇怪的參數頁或標籤頁，代表你的站內連結結構在製造垃圾路徑。

## 三站資料合併後的行動表

把三個來源整理成一張決策表：

- 0 點擊＋已索引＋主題重要 → **補真材料**（真實案例、價格區間、規格限制）。
- 0 點擊＋Crawled - currently not indexed＋主題和其他頁重複 → **合併＋301**。
- 0 點擊＋沒外鏈＋主題根本沒人搜 → **移除**。

三選一的完整決策流程見〈[0 點擊頁面該合併、刪除還是補料？](/zh-hant/blog/zero-click-pages/)〉。

## 多久跑一次？

**每季一次**。把上述流程做成固定的季度儀式：匯出 → 分類 → 處理 30 頁 → 記錄。連續跑 3–4 季後，多數網站的「Crawled - currently not indexed」數量會明顯下降，新文章的索引速度也會變快。

## 常見問題（FAQ）

**Q1：「Crawled - currently not indexed」的頁面會自己好嗎？**
少數會（Google 偶爾重新評估），但等待不是策略。這個狀態本質上是品質判決，不動內容它大概率一直躺在那裡。

**Q2：GSC 資料和 GA4 的流量數字對不上，該信哪個？**
找內容負債時信 GSC。GA4 統計所有管道的工作階段，GSC 才是「自然搜尋點擊」的第一手資料；兩者定義不同，數字本來就不會一致。

**Q3：新文章發布兩週就出現在「Discovered - currently not indexed」，算負債嗎？**
不算，這在新站很常見。給它 4–8 週，同時從高權重頁面（首頁、支柱頁）加內部連結拉抬爬取優先度。

**Q4：可以直接把 0 點擊頁全部 noindex 嗎？**
不建議一刀切。noindex 只是把頁面藏起來，沒有回收它的內部連結權重，也沒有解決內容品質問題。三選一（合併／移除／補料）才是根治。
