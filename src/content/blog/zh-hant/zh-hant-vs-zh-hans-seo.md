# 繁中 vs 簡中 SEO：不是字轉換那麼簡單

**繁體中文和簡體中文 SEO 是兩個市場的兩套工程，不是「把字轉一轉」的工作：兩邊的用詞不同（軟體 vs 軟件）、搜尋引擎生態不同（Google vs 百度）、關鍵字搜尋行為也不同。** 直接用工具做繁簡轉換上線，得到的不是第二個市場，而是一批內容負債。

## 為什麼「繁轉簡」會失敗？

很多團隊的簡中版是這樣誕生的：拿繁中文章跑 OpenCC 或機器轉換，五分鐘生出幾百頁。這種做法有三個致命傷：

1. **關鍵字直接錯位**：台灣人搜「軟體推薦」，大陸與海外簡中使用者搜「软件推荐」——不只字形不同，**詞根本不一樣**。轉換工具只換字形，不換詞，你的簡中頁面瞄準的其實是「不存在的關鍵字」。
2. **內容在地性為零**：範例、幣別、法規、平台（LINE vs 微信）全部對不上目標讀者的生活。
3. **重複內容風險**：對演算法而言，字轉換版與原版相似度極高，若 hreflang 再沒設好，繁簡互相蠶食幾乎是必然結局。

## 四大差異一次看懂

### 差異 1：用詞是兩套詞彙系統

| 概念 | 繁中（台灣） | 簡中（大陸／海外） |
|---|---|---|
| software | 軟體 | 软件 |
| program | 程式 | 程序 |
| search | 搜尋 | 搜索 |
| video | 影片 | 视频 |
| internet | 網路 | 网络 |
| data | 資料 | 数据 |

每一組差異都對應不同的搜尋量分布。做簡中 SEO 卻寫「搜尋引擎優化」，等於放棄了「搜索引擎优化」這個真正有量的詞。

### 差異 2：搜尋引擎生態完全不同

繁中市場（台灣、香港）以 Google 為絕對主流。簡中市場則要先分清楚你打的是哪一塊：**Google 自 2010 年退出中國大陸後在當地無法使用，大陸市場是百度為主的另一套玩法**——收錄機制、外鏈邏輯、備案要求都不同；而**海外簡中使用者**（馬來西亞、新加坡、北美華人）用的仍是 Google 與 Bing。多數國際站的簡中版，實際目標應設定為「海外簡中＋Google」，除非你真的要投入百度 SEO 的完整資源。

### 差異 3：關鍵字研究必須各自重做

正確做法是為簡中市場**重新跑一次完整的關鍵字研究與意圖矩陣**，而不是翻譯繁中的關鍵字清單。同一個主題，兩個市場的問法、關心的子題、競爭程度都可能不同——例如 SEO 工具類主題，簡中市場對某些工具的討論熱度與繁中市場差距很大，直接翻譯清單會做出一堆沒人搜的文章。

### 差異 4：E-E-A-T 與在地訊號

作者背景、案例、引用來源都需要在地化。給台灣讀者看的案例引用 momo 與 PChome 很自然，簡中讀者則對不同的平台生態有共鳴。內容「看起來像不像當地人寫的」，會直接反映在停留時間與互動訊號上。

## 繁簡雙軌的正確做法

1. **架構**：用子目錄 `/zh-hant/`、`/zh-hans/` 分流，每頁配好 hreflang（`zh-Hant`／`zh-Hans`＋x-default）。
2. **關鍵字**：兩個語言各自做關鍵字研究、各自建意圖矩陣。
3. **內容**：以繁中版為「素材」而非「原文」，由懂簡中市場用語的人（或經在地化校對的流程）重寫用詞、範例與內鏈錨文本。
4. **追蹤**：GSC 依 `/zh-hant/`、`/zh-hans/` 目錄分開看成效，各自迭代。

## 常見問題（FAQ）

**Q1：預算有限，繁簡只能先做一個，選哪個？**
選你的主力市場。以台灣客戶為主就先做深繁中，等繁中群集成形、有轉換證據後，再用同一套方法論複製到簡中——而不是兩邊各做一半。

**Q2：簡中版一定要放棄大陸市場嗎？**
不是放棄，是分開決策。大陸市場需要百度 SEO、ICP 備案與境內主機等完整投入，是另一個專案；海外簡中市場用 Google 就能觸及，成本低得多，適合作為簡中版的第一階段目標。

**Q3：機器翻譯完全不能用嗎？**
可以當草稿，不能當成品。機器轉換＋在地編輯校對用詞與範例，是成本與品質的合理平衡點；機器直出不校對，就是在製造 90 天後要回頭清理的內容負債。

**Q4：繁簡兩版的 title 和 meta description 也要分開寫嗎？**
要。title 是關鍵字瞄準的第一現場，「搜尋」和「搜索」的差異就發生在這裡；連 slug、圖片 alt 與 Schema 裡的文字都應該用該語言的在地用詞。
